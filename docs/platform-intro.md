# 晶格视界 · 数字文保 — 平台介绍

## 定位

晶格视界是一个面向历史文化街区活化保护的三维数字化平台。它以浏览器端 3D Gaussian Splatting 实时渲染为核心，构建"数字建档—态势总览—公众参与—管理审核"四位一体的技术体系，为基层文保单位、社区居民与管理决策者提供统一的数字协作空间。

---

## 技术路线

平台采用 **3D Gaussian Splatting**（3DGS）作为三维重建技术基础。该技术于 SIGGRAPH 2023 获评最佳论文，其核心优势在于：以消费级设备（智能手机）采集 100–200 张环绕照片，即可重建具有毫米级空间精度的三维场景，数据存储为公开规范的 `.splat` 格式（每高斯点 32 字节，含位置、尺度、颜色与旋转四通道）。相较传统激光扫描方案，硬件成本从十万级降至一部手机，操作门槛从专业测绘人员降至社区工作者。

渲染端基于 **WebGL 2.0** 实现纯前端实时渲染，无需后端 GPU 算力支持，部署仅需静态文件托管服务。存储层采用 **Supabase PostgreSQL** 云端协同架构，网络薄弱环境下自动降级至浏览器本地存储，保障乡镇级基层单位的正常使用。

---

## 核心功能

### 数字建档

每处文物建筑关联九维度元数据：保护级别、建造年代、建筑风格、保存状态、地理坐标、历史影像、文史档案、修缮记录、红色文化印记。字段设计参照《不可移动文物普查登记规范》，支持多条件组合筛选与全文检索。三维场景与结构化元数据双向关联——在模型中点击建筑，即可展开完整档案面板。

### 文保态势总览

以交互式地图呈现辖区内文保资源空间分布，颜色编码直观标示保存状态（完好/良好/需修缮/危急），为修缮资金优先级排序提供数据依据。支持按行政区划、保护级别、建筑年代等维度下钻分析。

### 公众参与机制

居民可通过平台提交文物损坏报告，包含现场文字描述与照片，选择上报类别后系统自动分配 8 位追踪编码。审核状态在平台上全程公开可查，将"最后一公里"的邻里口传转化为结构化、可追溯的数字流程。

### 审核管理

上报内容进入 pending → approved → rejected 标准状态机。管理员可在三维模型上精确定位标注，审核意见与空间坐标关联存储。平台同时支持管理员直接创建模型档案——通过结构化表单填录完整元数据（基本信息、文保信息、历史照片、文史档案、修缮记录、红色印记六个维度），适用于文物部门批量数字化存量档案的需求。

---

## 治理闭环

平台构建了"感知上报—信息汇聚—专业研判—行政响应—闭环反馈"五阶段操作链路：

1. **感知上报** — 居民发现建筑隐患，通过平台提交报告，系统生成追踪编码
2. **信息汇聚** — 报告进入审核队列，其他居民可补充信息，个体发现转化为群体共识
3. **专业研判** — 管理员审核确认后在三维模型上精确标注损伤位置
4. **行政响应** — 管理部门基于态势总览中的状态变化，安排修缮资金与施工计划
5. **闭环反馈** — 修缮完成后更新保存状态与修缮记录，上报者收到处置完成通知

全链路操作记录可审计，决策依据从逐级文字报告变为结构化数据与三维可视化证据。

## 技术指标

| 指标 | 数值 |
|------|------|
| 三维重建精度 | 毫米级（取决于采集密度） |
| 单模型高斯点数 | 75 万–300 万 |
| 渲染帧率 | ≥30 fps（中等硬件） |
| 数据格式 | `.splat`（32B/point，公开规范） |
| 采集设备要求 | 智能手机（100–200 张环绕照片） |
| 服务端依赖 | 无（纯前端渲染；审核功能需数据库） |

## 运行环境

- **浏览器**：Chrome 90+ / Edge 90+ / Firefox 120+（需 WebGL 2.0 支持）
- **桌面端**：Windows 10+ / macOS 11+
- **移动端**：iOS 15+ Safari / Android Chrome（功能完整，适配持续优化中）
- **部署**：静态文件托管（Vercel / GitHub Pages），零服务器运维
